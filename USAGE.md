# NotebookLLM 使用教程

## 📖 用户指南

### 第一步：启动应用

1. **启动后端** 
   - Windows: 双击 `run_backend.bat` 或运行 `python main.py`
   - 等待看到 "Uvicorn running on http://0.0.0.0:8000"

2. **启动前端**
   - Windows: 双击 `run_frontend.bat` 或运行 `python -m http.server 3000`
   - 浏览器访问 `http://localhost:3000`

### 第二步：准备内容

有两种方式提供输入内容：

#### 📄 上传文件
- 支持格式：TXT、DOCX、PDF
- 方法：拖拽到上传区或点击选择文件
- 提示：在"上传方式"选择"上传文件"

#### ✍️ 直接输入文本
- 方法：选择"直接输入文本"，在文本框中粘贴内容
- 适合：临时笔记、快速测试

### 第三步：配置讨论参数

在右侧面板配置以下参数：

#### 1. **话题深度**
- **浅层讨论** - 适合快速概览，简洁明了
- **中等深度** - 平衡分析，推荐选择
- **深层讨论** - 详细探讨，适合学术讨论

#### 2. **对话长度**
- **短篇** (5-10分钟) - 适合快速讨论
- **中等** (10-20分钟) - 最平衡的选择
- **长篇** (20-30分钟) - 深入讨论和教学

#### 3. **语气风格**
- **专业严肃** - 学术、商务场景
- **轻松随意** - 日常讨论、娱乐内容
- **教育启蒙** - 教学讲解、知识传递
- **娱乐幽默** - 有趣、引人入胜的讨论

#### 4. **讲者名称**
- 默认：讲者1、讲者2
- 自定义：输入您喜欢的名称（如"张教授"、"李学生"）

### 第四步：生成讨论

点击 **🚀 生成讨论** 按钮

- 等待提示信息：正在生成讨论内容，请稍候...
- 生成时间：通常30秒到2分钟
- 完成后：讨论内容将显示在底部面板

## 💡 使用技巧

### 最佳实践

1. **内容质量**
   - 提供清晰、完整的输入内容
   - 避免过短的内容（至少50字）
   - 避免过长的内容（最好不超过5000字）

2. **参数组合**
   - **学习讨论** → 中等深度 + 教育启蒙 + 中等长度
   - **学术讨论** → 深层讨论 + 专业严肃 + 长篇
   - **快速了解** → 浅层讨论 + 轻松随意 + 短篇
   - **娱乐内容** → 中等深度 + 娱乐幽默 + 中等长度

3. **讲者设置**
   - 使用有意义的名称增加沉浸感
   - 例如：教授/学生、主持人/嘉宾、专家/提问者

### 常用场景

## 🔧 高级功能

### 批量处理

1. 上传多个文件时，逐个处理
2. 为每个文件调整不同的参数
3. 保存好每次的结果

### 导出结果

讨论内容生成后：
1. 全选讨论内容 (Ctrl+A)
2. 复制 (Ctrl+C)
3. 粘贴到文本编辑器、Word或Markdown工具中

### 自定义样式

编辑 `frontend/index.html`，修改CSS颜色：

```css
/* 更改主色 */
--primary-color: #667eea;  /* 修改为您喜欢的颜色 */

/* 更改背景渐变 */
background: linear-gradient(135deg, #YOUR_COLOR1 0%, #YOUR_COLOR2 100%);
```

## ❌ 常见错误与解决

| 错误信息 | 原因 | 解决方案 |
|---------|------|--------|
| 后端连接失败 | 后端未启动 | 确保运行了 `python main.py` |
| API密钥无效 | 密钥错误或过期 | 检查 API_KEY 配置 |
| 内容为空 | 未输入或上传内容 | 先上传文件或输入文本 |
| 生成超时 | API响应慢或网络问题 | 重试或换用faster模型 |
| 讨论格式错误 | LLM返回格式异常 | 重新生成或检查输入内容 |

## 📈 性能和限制

### 响应时间

- GPT-3.5-turbo: 30-60秒
- GPT-4: 60-120秒
- 网络延迟：可能额外增加10-30秒

### 内容限制

- 最大输入长度：~5000字符
- 最大输出长度：~4000字符
- 建议：不要过长，质量优于数量

## 🆘 获取帮助

### 常见问题

**Q: 为什么生成速度这么慢？**
A: OpenAI API响应时间较长。使用gpt-3.5-turbo会更快。

**Q: 可以生成多少对讨论？**
A: 一次生成一个讨论对（speaker1 + speaker2）。多个话题需多次生成。

**Q: 生成的讨论质量怎样？**
A: 取决于输入内容质量。详细、清晰的输入会产生更好的讨论。

**Q: 可以离线使用吗？**
A: 不行，需要调用OpenAI API。需要互联网连接。

**Q: 支持其他语言吗？**
A: 支持！后端兼容多语言。试试用英文、日文等输入。

### 技术支持

- 检查浏览器控制台错误：F12
- 查看后端日志：运行后端的终端窗口
- API文档：访问 http://localhost:8000/docs
